# SumoNoteBook 三層資料夾架構總覽

SumoNoteBook 採用「衣櫃式三層資料夾架構」，模擬真實的筆記本分類方式，讓知識管理更有條理。

## 🗂️ 三層架構

```
SumoNoteBook/
├── 📥 Raw (原始資料層)
│   ├── sources/       # 來源資料（網頁、PDF、文件）
│   ├── raw_notes/     # 未整理的筆記
│   ├── processed/     # 處理中的資料
│   └── ...
│
├── 📚 Wiki (結構化知識層)
│   ├── concepts/      # 概念定義
│   ├── summaries/     # 重點摘要
│   ├── MOC.md         # 主題地圖
│   └── README.md      # 本層說明
│
└── 🏛️ Palace (記憶宮殿層)
    ├── closet/        # 衣櫃 - 依情境分類的知識
    ├── drawer/        # 抽屜 - 草稿與工作區
    ├── learning/      # 學習記錄
    └── memory/        # 長期記憶
```

## 層級說明

### 📥 Raw 層 - 原始資料庫

**用途**：存放未經處理的原始資料
- 網頁擷取、PDF、文檔
- 隨手記下的筆記
- 來源連結

**特性**：
- 資料量大但未結構化
- 可能包含重複或雜訊
- 需要進一步處理

---

### 📚 Wiki 層 - 結構化知識庫

**用途**：存放經過整理的結構化知識
- 概念定義與說明
- 文件摘要與重點
- 主題之間的連結

**特性**：
- 經過整理、去蕪存菁
- 可快速查找
- 蘇茉家族的共享知識庫

---

### 🏛️ Palace 層 - 記憶宮殿

**用途**：個人化、高價值的知識資產
- 情境分類（正式/休閒/工作）
- 長期記憶與經驗
- 學習成長記錄

**特性**：
- 高度個人化
- 與蘇茉身份綁定
- 持續累積的智慧結晶

## 資料流向

```
來源輸入 → Raw (暫存) → Wiki (整理) → Palace (內化)
                      ↓
               搜尋與查找
```

## 使用時機

| 層級 | 使用時機 |
|------|----------|
| Raw | 剛收集到的新資料、待處理的來源 |
| Wiki | 需要查找已整理的知識、概念定義 |
| Palace | 建立個人知識體系、情境化應用 |

## 設計原則

1. **就近原則**：資料放在最常用的位置
2. **整理原則**：定期將 Raw 移轉到 Wiki
3. **內化原則**：重要知識沉淀到 Palace
4. **搜尋優先**：所有層都可以被搜尋

---

*這個架構參考自盧曼卡片盒（Zettelkasten）與傳統筆記本的概念，讓蘇茉家族的知識管理更有系統。*
