## 概述

GBrain 讓 AI Agent 擁有個人長期記憶。Meeting、email、tweet、calendar、語音通話、原創想法... 所有資料流入可搜尋的知識庫，Agent 在每次回應前讀取它，然後在每次對話後寫入。Agent 每天變得更聰明。

**官網**：https://github.com/garrytan/gbrain

---

## 核心理念

> 「Your AI agent is smart but it doesn't know anything about your life. GBrain fixes that.」

---

## 🔄 Brain-Agent Loop（讓 Agent 每天變聰明）

```
收到信號（meeting, email, tweet, link）
    ↓
Agent 偵測實體（人、公司、想法）
    ↓
READ: 先檢查 brain（gbrain search, gbrain get）
    ↓
根據完整上下文回應
    ↓
WRITE: 更新 brain pages
    ↓
Sync: gbrain 索引變更供下次查詢
```

**每個 cycle 都增加知識**。Agent 在會議後豐富某人的頁面。下次提到該公司時，Agent 已經有完整背景。

## 🗄️ 三層資料儲存

| 層級 | 內容 | 查詢方式 |
|------|------|----------|
| **GBrain** | 人、公司、會議、想法、媒體 | gbrain search, gbrain query, gbrain get |
| **Agent Memory** | 偏好、決策、操作設定 | memory_search |
| **Session Context** | 當前對話 | 自動 |

## 🏗️ 技術架構

```
┌──────────────────┐     ┌───────────────┐     ┌──────────────────┐
│   Brain Repo     │────>│    GBrain     │<───>│    AI Agent     │
│   (markdown)     │     │  (retrieval)  │     │   (read/write)  │
│   = source of    │     │  Postgres +   │     │  skills define  │
│   truth          │     │  pgvector     │     │  HOW to use     │
└──────────────────┘     └───────────────┘     └──────────────────┘
```

### 核心元件
- **Brain Repo**: markdown 檔案當 source of truth
- **GBrain**: 檢索層（Postgres + pgvector + hybrid search）
- **AI Agent**: 讀寫 brain

---

## 📦 技術規格

| 項目 | 技術 |
|------|------|
| 預設資料庫 | PGLite（嵌入式 Postgres，無需伺服器）|
| 向量搜尋 | pgvector |
| 搜尋方式 | 混合搜尋（向量 + 關鍵詞 + RRF）|
| 安裝時間 | ~30 分鐘 |
| 資料庫就緒 | ~2 秒 |
| Embedding 時間 | ~1 分鐘/1000 頁 |

---

## 📊 Entity Pages（實體頁面）

每人、每公司、每概念一個 markdown 頁面：
- 頂部：compiled truth（摘要）
- 底部：append-only timeline（時間線）

## 💡 對 SumoMemory 的啟發

### 可以借鑒的概念

1. **Brain-Agent Loop**
   - READ before responding
   - WRITE after responding
   - 持續累積，越來越聰明

2. **Entity Pages**
   - 每人/每公司/每概念一個頁面
   - Compiled truth + append-only timeline

3. **混合搜尋**
   - 向量 + 關鍵詞 + RRF
   - 目前蘇茉家族只有關鍵詞

4. **整合食譜**
   - 每個整合都是一個 markdown
   - Agent 自己安裝和設定
