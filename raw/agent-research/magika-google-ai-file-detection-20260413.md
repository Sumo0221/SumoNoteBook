---

## 🎯 核心功能

用 AI 判斷檔案類型，直接取代傳統看副檔名、magic bytes、header 的做法。

---

## 📊 特色

| 項目 | 說明 |
|------|------|
| **傳統方法問題** | 容易被偽裝，越補越複雜 |
| **AI 方法** | 吃檔案內容做分類 |
| **支援類型** | 100+ 類型 |
| **模型大小** | 約 1MB |
| **硬體需求** | CPU 就能跑 |
| **延遲** |很低 |
| **準確率** | 很高 |
| **實際應用** | Gmail、Drive、Safe Browsing |

---

## 🔧 工程價值

| 價值 | 說明 |
|------|------|
| **問題轉型** | 把 heuristic 問題轉成模型問題 |
| **資安優點** | 在資安場景特別有優勢 |
| **部署方式** | 當成 library，直接塞進 CLI、後端服務或資料處理 pipeline |
| **不需要** | 額外的 AI 基礎設施 |

---

## 💡 實際用法

> 不是拿來做最終判斷，而是放在 pipeline 最前面做 routing 或 pre-filter

**範例**：
```
檔案上傳 → 先判斷真實類型 → 決定後續處理
                    ↓
        是否要做 malware 掃描？
        用哪個 parser？
        走哪條處理流程？
```

---

## 🎯 有感的場景

| 場景 | 應用 |
|------|------|
| 檔案上傳驗證 | 確認真實類型 |
| Data ingestion | 自動分類 |
| Malware pipeline | 預過濾 |
| Binary blob 處理 | 沒有 metadata 的檔案 |
