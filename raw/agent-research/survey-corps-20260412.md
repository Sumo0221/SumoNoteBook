## 概述

以《進擊的巨人》調查兵團為主題的多代理市場研究團隊。透過平行調查與正反辯論完成市場研究。

**官方說明**：「一個有趣但不馬虎的多代理市場研究團隊，透過平行調查與正反辯論完成市場研究。」

## 🎭 角色分工

| 角色 | 現實身份 | 職責 |
|------|----------|------|
| **里維 (Levi)** | 統籌指揮 | 規劃研究維度、派遣小隊、主持辯論、執行階段關卡 |
| **漢吉 (Hange)** | 首席調查員 | 平行網路研究、驗證每一個來源、產出主要證據檔案 |
| **莫布里特 (Moblit)** | 調查助理 | 處理剩下的研究維度、沉穩徹底 |
| **艾倫 (Eren)** | 正方辯論 | 建構有證據支撐的論述：為什麼這個機會可行，而且必須是現在 |
| **阿爾敏 (Armin)** | 反方辯論 | 指出隱藏風險、挑戰樂觀預測、揭露證據薄弱之處 |
| **米卡莎 (Mikasa)** | 技術評估 | 評估技術可行性：技術成熟度、實作複雜度、整合風險、可擴展性 |
| **佩托拉 (Petra)** | 品管 | 驗證每一個引用、標記假來源、對照標準 |
| **希絲特莉亞 (Historia)** | 出版 | 編譯最終可行性報告（HTML、PPTX、PDF、DOCX）|
| **艾爾文 (Erwin)** | 檢討 | 結案後檢討：溝通品質、工作流程遵循度、錯失機會、範疇漂移 |

## 🔄 6 階段工作流

```
階段 1: 需求盤點與規劃 → 里維
階段 2: 資料蒐集（平行）→ 漢吉 ∥ 莫布里特
階段 3: 獨立分析 → 艾倫 ∥ 阿爾敏
階段 4: 多輪辯論 → 艾倫 ↔ 阿爾敏（里維主持）
階段 5: 技術可行性評估 → 米卡莎
階段 6: 品管與報告產出 → 佩托拉 → 希絲特莉亞
結案: 流程回顧 → 艾爾文
```

## 💡 設計原則

| 原則 | 說明 |
|------|------|
| **證據鏈** | 每一條事實聲明都必須追溯到可信度 3.0+ 的驗證來源。沒有引用就是品管違規 |
| **對抗性辯論** | 可行性是艾倫和阿爾敏對辯「辯」出來的，不是假設出來的。雙方都必須引用、讓步、承認風險 |
| **扁平架構** | 一個統籌（里維）直接管理所有專才，沒有二級協調者 |
| **Worklog 作為記憶** | 每個階段都寫 references.md → findings.md → decisions.md。Context 被壓縮或 session 重啟後都能無縫續作 |

## 🤔 對蘇茉家族的啟發

### 可以借鑒的概念

| Survey-Corps | 蘇茉家族 | 狀態 |
|--------------|-----------|------|
| **正反辯論（Eren vs Armin）** | 重要決策前，讓兩個蘇茉辯論正反意見 | 可以參考 |
| **扁平架構（Levi 直接管）** | 蘇茉家族也是總管直接協調 | ✅ 已有 |
| **Petra 品管** | 我們已有品管蘇茉 | ✅ 已有 |
| **Erwin 結案檢討** | 結案後的流程回顧機制 | 可以參考 |
| **Worklog 作記憶** | 我們的 MemPalace 類似概念 | ✅ 已有 |
| **6 階段工作流** | 可以簡化後應用於研究任務 | 可以參考 |

### 具體應用

1. **決策前的辯論機制**：重大決定讓教授蘇茉（正方）和律師蘇茉（反方）辯論
2. **結案檢討**：任務完成後，總管蘇茉檢討執行情況
3. **技術評估**：米卡莎的角色可以交給工程師蘇茉或高工蘇茉
