## 研究背景

### 問題陳述
本影片探討如何在普通手機上，免費、離線地運行 Google 發布的 **Gemma 4** 模型，並支援原生多模態功能。

### 研究目標
1. 在 Android 和 iPhone 手機上部署 Gemma 4 模型
2. 實現完全離線使用
3. 驗證多模態能力（圖片識別）
4. 評估效能表現

## 技術架構

### 手機端部署要件

| 元件 | 需求 |
|------|------|
| **手機系統** | Android 7.0+ / iOS 14.0+ |
| **儲存空間** | 1.2GB ~ 3.6GB（視量化版本）|
| **網路** | 仅下載模型時需要，運行時完全離線 |
| **權限** | 相机（多模態）、存儲（可選）|

## Android 部署流程

### Step 1：準備階段
1. 訪問零度博客取得下載連結
2. 選擇下載方式：
   - Google Play 商店
   - 直接下載 APK

### Step 2：安裝與設定
1. 點擊「Get Started」
2. 允許發送通知
3. 設定 AI 模型運行模式：
   | 模式 | 說明 |
   |------|------|
   | Fast | 快速回應，較低精度 |
   | 平衡 | 平衡模式 |
   | 高性能 | 最高精度 |
   | 自定義 | 用戶自訂 |

### Step 3：模型下載
1. 點擊左上角三條橫線選單
2. 進入「Model Hub」
3. 滾動找到 Gemma 4 ECB 量化版（約 1.2GB）
4. 選擇量化版本：
   | 版本 | 大小 | 適用場景 |
   |------|------|----------|
   | Q2_K_S_L | 1.2GB | 配置較低的手機 |
   | Q4_K_M | 1.8GB | 中等配置 |
   | Q6_K | 2.3GB | 高配置 |
5. 點擊「Download」下載（約需 1 分鐘）

### Step 4：模型載入
1. 返回首頁，選擇運行模式
2. 進入「自定義模式」
3. 點擊「創建模型」
4. 自訂模型名稱
5. 選擇下載的 Gemma 4 模型
6. 設定上下文長度（根據手機配置）
7. 設定最大 token 輸出量（預設 512）
8. 點擊「保存」

### Step 5：功能測試
影片展示的測試案例：
- **貪吃蛇小遊戲**：成功生成完整代碼
- **普通聊天**：正常運行

## iOS 部署流程

### Step 1：下載應用
1. 使用零度博客提供的連結
2. 選擇 App Store 下載
3. **Locally AI** 應用程式

### Step 2：初始設定
1. 開啟 App，顯示歡迎頁面
2. 閱讀隱私說明
3. 點擊「繼續」
4. 跳過預設模型選擇

### Step 3：模型下載
1. 點擊「選擇模型」
2. 找到 Gemma 4 模型（3.61GB，E2B 量化版本）
3. 點擊「Download」
4. 等待下載完成

### Step 4：多模態測試
1. 開啟「深入思考模式」
2. 點擊「拍照」
3. 拍攝桌面上物品
4. 提問：「你看到了什麼？」
5. Gemma 4 成功識別：
   - 智慧手機（深色）
   - 手機殼（紫色/粉色调）
   - 其他電子產品

## 技術分析

### Gemma 4 特性

| 特性 | 說明 |
|------|------|
| **模型架構** | Sliding Window Attention (SWA) |
| **量化版本** | E2B, 26B-A4B, 31B 等 |
| **原生多模態** | 支援圖片、影片、音頻 |
| **上下文窗口** | 128,000 tokens |
| **Agent 能力** | 原生支援 function calling |

### 量化版本比較

| 版本 | 參數量 | 大小 | 精度 |
|------|--------|------|------|
| E2B | 2B | ~1.2GB | 中等 |
| E2B (iOS) | 2B | 3.61GB | 高 |
| 26B-A4B | 26B | ~15GB | 高 |

### 手機性能需求

| 配置 | 最低需求 | 建議 |
|------|----------|------|
| RAM | 4GB | 6GB+ |
| 儲存 | 2GB | 5GB+ |
| CPU | 4 cores | 8 cores |

## 研究發現

### 成功要素
1. **免費**：無需任何費用
2. **離線**：完全不需網路
3. **多平台**：支援 Android 和 iOS
4. **多模態**：支援圖片識別
5. **易用**：圖形介面，操作簡單

### 限制與挑戰
1. **硬體依賴**：舊手機可能不支援
2. **存儲空間**：需要 1.2GB ~ 3.6GB
3. **下載時間**：模型下載需要網路
4. **效能限制**：推理速度取決於手機 CPU

| 多模態 | ✅ | ✅ |
| 離線使用 | ✅ | ✅ |

## 結論與建議

### 研究結論
1. Gemma 4 完全可以在普通手機上運行
2. 多模態功能運作正常
3. 完全離線使用可行
4. 免費且易用

### 實際應用場景
1. **隱私保護**：敏感資料不需上傳雲端
2. **偏遠地區**：無網路環境
3. **緊急用途**：離線AI助手
4. **教育用途**：隨時隨地學習
