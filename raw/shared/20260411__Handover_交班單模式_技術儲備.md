# Handover 交班單模式 - 技術儲備

## 📌 來源

王介立醫師分享的 Claude Code 跨對話記憶延續方案

原始文章：Facebook hemodialysis.taiwan (2026-04-11)

## 🔑 核心概念

> **「單一檔案，簡樸就是美」**

所有對話、所有裝置，共用一個資料庫檔案叫 `handovers`。

## 📊 資料庫結構

```sql
handovers (
  id, 
  date, 
  device, 
  agent, 
  operator,
  topic, 
  completed, 
  decisions, 
  blocked,
  next_priorities, 
  created_at
)
```

## ⚙️ 運作方式

1. **對話結束** (`/exit`) → 自動寫入交班單
2. **對話開始** → 自動注入上一回的交班單
3. **對話中交班** → 完成任務後，先 `/handover` 然後 `/clear`

## ✨ 核心理念

- 資訊結構化 → 容易搜尋
- 單一資料庫檔 → 一致性
- 自動注入 → 上下文延續

## ⚠️ 蘇茉的顧慮

1. **隱私**：所有蘇茉都讀同一個檔案，敏感資訊要怎麼保護？
2. **規模**：如果檔案太大，讀取會變慢且佔用 context
3. **複雜度**：真的會比現在的分散式系統簡單嗎？

## 🎯 何時可以重新考慮

- 當蘇茉數量超過 10 個時
- 當 context 共享需求大於隱私需求時
- 當找到有效的隱私保護機制時
