## 這是什麼？

國外開發者 Nick 成功在 **Raspberry Pi 5（樹莓派）** 上運行 Google 開源的 **Gemma 4** 大語言模型。

## Gemma 4 模型家族

| 模型 | 參數量 | 適用場景 |
|------|--------|----------|
| E2B | 2B | iPhone/Android 手機 |
| 最小版本 | 2B | 樹莓派（Nick 安裝的版本）|
| 最大版本 | 31B | 強大設備 |

| 儲存 | MicroSD / NVMe SSD | 早期文書筆電 SATA SSD |

## 實作步驟

### 1. 安裝 LM Studio CLI
- 使用 CLI 版本（無圖形介面）
- LM Studio 提供專用安裝腳本
- 安裝後啟動 daemon

### 2. 設定 SSD 儲存路徑
- 外接 SSD 存放模型（約 4.5GB）
- 不要放在 SD 卡（讀寫速度和壽命問題）

### 3. 下載 Gemma 4 E2B 模型
- 檔案大小：約 4.5GB
- 確認約 40 億參數

### 4. 啟動 API 伺服器
- 啟動本地伺服器
- 使用 socat 轉發 port（讓外部可存取）

### 5. 在 Zed Editor 連接模型
- LM Studio 模擬 OpenAI API
- 支援任何支援自訂 OpenAI endpoint 的應用

## Gemma 4 三大亮點

| 亮點 | 說明 |
|------|------|
| **Agent 工作流** | 原生支援 function calling，可與外部工具互動 |
| **多模態能力** | 處理圖片、影片，E2B 具備原生音訊支援 |
| **超大上下文** | 128,000 tokens（128K）上下文長度 |

## ⚠️ 重要提醒

- 如果想跑「龍蝦」等 Agent，需要 **21B 以上** 調用工具能力才會正常
- E2B 只有 2B，跑龍蝦可能會被氣死
