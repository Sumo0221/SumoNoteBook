## 2. 優化建議

### 2.1 Workspace 隔離強化

| 建議 | 優先級 | 說明 |
|------|--------|------|
| 嚴格目錄權限 | 高 | 各 workspace 設定唯讀隔離，防止意外污染 |
| 統一 temp 管理 | 高 | 所有臨時檔案統一至 `workspace/temp`，每日 02:00 清除 |
| 建立對照表文件 | 中 | 在各 workspace 維護 `WORKSPACE_MAP.md` |

### 2.2 Cron 路由標準化

```
┌─────────────────────────────────────────────────────────┐
│                   Cron 路由規則                         │
├─────────────────────────────────────────────────────────┤
│  sessionTarget = "main"      (systemEvent jobs)         │
│  sessionTarget = "isolated"  (Owner 前綴 tasks)         │
│  sessionTarget = "main"      (一般排程)                 │
└─────────────────────────────────────────────────────────┘
```

**建議**：
- 在 OpenClaw 設定檔中明確定義 `cron.target` 映射
- Owner 前綴作為任務標記，不改變 sessionTarget 行為

### 2.3 架構改進方案

#### 方案 A：集中式協調（推薦）
```
┌──────────────┐
│  main       │  ← 協調者，負責任務分派
└──────┬──────┘
       │ 任務路由
       ▼
┌──────────────┐     ┌──────────────┐
│  isolated   │ ←── │ 其他蘇茉     │
│  (Owner)    │     │ (專業分工)   │
└──────────────┘     └──────────────┘
```

**優點**：
- 簡化路由邏輯
- 便於監控與除錯
- main session 掌握全域視角

**缺點**：
- main 可能成為瓶頸
- 需要 main 高可用

#### 方案 B：分散式協調
```
┌──────────────┐
│  main       │  ← 協調者（不做具體任務）
└──────┬──────┘
       │ 請求轉遞
       ▼
┌──────────────┐     ┌──────────────┐
│  蘇茉 A      │ ←── │ 蘇茉 B       │
│  (isolated)  │     │ (isolated)   │
└──────────────┘     └──────────────┘
```

**優點**：
- 無單點瓶頸
- 各蘇茉完全獨立

**缺點**：
- 跨蘇茉協作複雜
- 路由邏輯分散

### 2.4 安全性建議

| 項目 | 說明 |
|------|------|
| API Key 管理 | 集中至 main，跨蘇茉呼叫需透過安全通道 |
| 敏感資料 | 各 workspace 隔離，嚴禁明文儲存 |
| 存取日誌 | 統一收集各蘇茉操作日誌至 main |
