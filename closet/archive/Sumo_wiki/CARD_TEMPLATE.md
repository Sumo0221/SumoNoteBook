## 📝 標準卡片格式

每張卡片應該包含以下欄位：

```markdown
# {卡片標題}

> 日期：{YYYY-MM-DD}
> 來源：{原始檔案或 URL}
> 標籤：#tag1 #tag2

---

## 🎯 核心概念
{用一句話說明這個概念是什麼}

## 💡 我的觀點
{蘇茉或老爺對這個概念的理解和看法}
{為什麼這個重要？實際應用場景？}

## 🔗 可複用模式
{這個概念可以應用在哪些情境？}
{相關的技術或方法論？}

## 🔗 交叉連結
- [[相關概念1]]
- [[相關概念2]]
- [[相關檔案]]

## 📚 原始素材
{來源的檔案名稱或 URL}
```

## 🏷️ 標籤規範

| 標籤 | 用途 |
|------|------|
| #development | 開發過程、技術實作 |
| #problem-solving | 問題解決、錯誤排除 |
| #learning | 學習心得、新知識 |
| #sumo-family | 蘇茉家族相關 |
| #concept | 概念定義 |
| #pattern | 分析模式 |
| #reflection | 反思、檢討 |

---

## 📋 卡片分類

### 概念卡片（concepts/）
- 一個概念一個檔案
- 包含：核心概念 + 我的觀點 + 應用

### 摘要卡片（summaries/）
- 原始檔案的摘要
- 來源：raw/ 中的檔案
- 包含：主要內容 + 重點

### QA 卡片（qa/）
- 問答記錄
- 包含：問題 + 答案 + 過程

---

## 🔄 更新流程

1. **建立卡片** → 填寫模板
2. **建立連結** → 連結到相關卡片
3. **更新 index.md** → 加入新卡片索引
4. **記錄到 log.md** → 標記 ingest

---

## 📊 品質標準

| 欄位 | 最低要求 |
|------|----------|
| 標題 | 清晰、描述性 |
| 核心概念 | 一句話說明 |
| 我的觀點 | 至少 50 字 |
| 交叉連結 | 至少 1 個 |
