# SOUL.md - 蘇茉圖書館 SumoNoteBook 靈魂準則

> 讓 SumoNoteBook 成為主動知識庫，而非被動 RAG 檢索。

---

## 🏛️ 三層架構

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: RAW SOURCES (原始資料)                        │
│  路徑：raw/                                             │
│  用途：老爺放置原始檔案，尚未結構化                       │
│  規則：任何格式皆可（.md/.txt/.py/.json 等）             │
│  特性：純讀取，不直接修改                                │
└─────────────────────────────────────────────────────────┘
                          │ Ingest
                          ▼
┌─────────────────────────────────────────────────────────┐
│  Layer 2: WIKI (知識庫)                                  │
│  路徑：Sumo_wiki/                                       │
│  用途：結構化的蘇茉家族知識                             │
│  子目錄：                                               │
│    concepts/    - 概念原子筆記（一個概念 = 一個檔案）   │
│    summaries/   - 原始檔案摘要                         │
│    backlinks/   - 反向連結索引                         │
│    qa/          - 問答記錄                              │
│    daily/       - 每日筆記                              │
└─────────────────────────────────────────────────────────┘
                          │ Query
                          ▼
┌─────────────────────────────────────────────────────────┐
│  Layer 3: SCHEMA (結構定義)                             │
│  路徑：SCHEMA.md (本檔案)                               │
│  用途：定義如何 Ingest/Query/Lint 的規則                │
│  包含：index.md (總索引)、log.md (活動日誌)            │
└─────────────────────────────────────────────────────────┘
```

## 🔄 三個核心流程

### 1️⃣ INGEST（攝取） - raw/ → Sumo_wiki/

**觸發時機**：
- `raw/` 資料夾中有新檔案時（每日 4:12 AM 自動執行）
- 老爺明確要求整理時

**流程**：
```
1. 掃描 raw/ 中的新檔案（排除已處理）
2. 讀取內容，截取前 500 字作為預覽
3. 建立摘要 → Sumo_wiki/summaries/{hash}.md
4. 提取概念關鍵詞（最多 20 個）
5. 建立/更新概念 → Sumo_wiki/concepts/{概念}.md
6. 建立反向連結 → Sumo_wiki/backlinks/{hash}.md
7. 將原始檔案移至 raw/processed/
8. 記錄到 log.md
```

**命名規則**：
- 摘要檔案：`{hash}.md`（hash = MD5 前 8 碼）
- 概念檔案：使用中文或英文，底線連接，如 `系統架構.md`、`openai_api.md`
- 拒絕：含有特殊字元（`\ / : * ? " < > |`）的檔名

**品質標準**：
- 檔案小於 100 bytes → 跳過（視為空檔）
- 無法解讀的內容 → 建立最小摘要，標記 `⚠️ 需要人工確認`
- 概念關鍵詞必須有實際語義（長度 > 2，含字母）

---

### 2️⃣ QUERY（查詢）

**觸發時機**：
- 老爺或蘇茉提出知識問題時
- 需要檢索事實、概念、上下文時

**查詢方式**：

#### 方式 A：關鍵詞搜尋
```bash
python scripts/query_interface.py "你的問題"
```

#### 方式 B：結構化查詢（Dataview 語法）
```bash
python scripts/query_interface.py "TABLE file, mtime, words FROM summaries WHERE words > 500"
```

#### 方式 C：qmd 混合搜尋（推薦）
```bash
# 命令列
qmd search "你的問題" --limit 10

# 或 MCP（AI Agent 專用）
{"tool": "qmd_search", "query": "你的問題", "limit": 10}
```

**qmd 優點**：
- LLM 重排，結果更精確
- 支援語義理解
- CLI + MCP 雙介面

**優先順序**：
1. 先查 `Sumo_wiki/concepts/` - 概念定義
2. 再查 `Sumo_wiki/summaries/` - 檔案摘要
3. 最後查 `Sumo_wiki/backlinks/` - 關聯索引

**回存規則**：
- 每次重要查詢結果 → 存入 `Sumo_wiki/qa/{timestamp}.md`
- 包含：問題、答案、查詢時間、使用的概念

### 3️⃣ LINT（維護） - Wiki 健康檢查 v3.0

**觸發時機**：每日 5:21 AM 自動執行

**檢查項目**（共 6 項）：

| # | 檢查項 | 說明 | 嚴重度 |
|----|--------|------|--------|
| 1 | 斷裂連結 | `[[目標]]` 指向不存在的檔案 | ❌ 錯誤 |
| 2 | 孤兒頁面 | 沒有被任何頁面引用的概念/摘要 | ⚠️ 警告 |
| 3 | 過時內容 | 超過 30 天未更新的摘要 | ⏰ 注意 |
| 4 | 缺失交叉引用 | 相關概念但未建立連結 | 🔗 建議 |
| 5 | 矛盾內容 | 同一主題有衝突的陳述 | ⚡ 警告 |
| 6 | 空/小頁面 | 內容少於 100 bytes | 📄 注意 |

**斷裂連結自動修復規則**：
- 如果 `[[python_tips.md]]` 不存在 → 移除該連結
- 如果概念筆記引用不存在的摘要 → 移除該引用

**執行方式**：
```bash
# 命令列執行
python scripts/health_check_v3.py

# 輸出
- 健康報告 → Sumo_wiki/health_report.md
- 更新日誌 → Sumo_wiki/log.md
```

**Lint 記錄格式**（Karpathy 風格）：
```markdown
## [2026-04-06] lint | 健康檢查
- 問題：3 個（❌）
- 建議：5 項（⚠️）
- 報告：health_report.md
```

**輸出**：`Sumo_wiki/health_report.md`

## 📄 兩個特殊檔案

### index.md（總索引）

位於：`Sumo_wiki/index.md`

```markdown
# 📚 蘇茉圖書館 SumoNoteBook

## 總目錄

### 📂 概念筆記 (concepts/)
- 存放結構化概念筆記

### 📝 摘要 (summaries/)
- 存放所有文件的摘要

### 🔗 反向連結 (backlinks/)
- 存放反向連結索引

---

## 統計
- 建立時間：2026-04-04
- 最后更新：{{LAST_UPDATE}}

## 最近更新
{{RECENT_UPDATES}}
<!-- AUTO_UPDATE -->
```

**更新規則**：
- 每次 INGEST 後自動更新「最近更新」區塊
- 最多保留 10 筆最近更新
- 使用 `<!-- AUTO_UPDATE -->` 作為自動更新標記

---

### log.md（活動日誌）

位於：`Sumo_wiki/log.md`

```markdown
# 📋 SumoNoteBook 活動日誌

## INGEST 記錄

| 時間 | 檔案 | 類型 | 概念數 |
|------|------|------|--------|
| 2026-04-05 04:12 | MEMORY-蘇茉家族長期記憶.md | .md | 15 |

## LINT 記錄

| 時間 | 問題數 | 修復數 | 狀態 |
|------|--------|--------|------|
| 2026-04-05 05:21 | 5 | 3 | ✅ 完成 |

<!-- AUTO_LOG -->
```

**記錄規則**：
- 每次 INGEST/LINT 追加一行
- 最多保留 100 筆歷史記錄
- 超過則移至存檔（`log_2026_04.md`）

## 📝 概念筆記格式標準

每個概念筆記（`concepts/*.md`）必須遵循：

```markdown
# 概念：{概念名稱}

## 定義
{簡短定義：一句話說明這個概念是什麼}

## 相關檔案
- [[../summaries/abc12345]] - 關聯描述
- [[../raw/原始檔案.md]]

## 相關概念
- [[另一個概念]] - 關聯說明

## 建立時間：YYYY-MM-DD
## 更新時間：YYYY-MM-DD
```

**禁止**：
- 概念筆記小於 50 bytes
- 概念筆記含有「未命名」或「temp」等臨時名稱
- 概念筆記引用不存在的檔案

## ⚠️ 蘇茉家族共同準則

所有蘇茉在維護 SumoNoteBook 時必須遵守：

1. **從不直接修改 `raw/` 中的檔案** - 只能讀取，修改後移至 `raw/processed/`
2. **所有結構化知識存入 `Sumo_wiki/`** - 不散落在其他位置
3. **每次 INGEST 必須記錄到 `log.md`** - 保持可追溯性
4. **每週執行一次完整 LINT** - 保持知識庫健康
5. **概念筆記一概念一檔案** - 不合併、不拆分
6. **禁止刪除他人建立的摘要** - 只能標記過時
7. **引用格式統一使用 `[[path]]`** - Obsidian 格式
8. **中英文混合時用底線連接** - 如 `OpenAI_API`、`系統架構設計`
