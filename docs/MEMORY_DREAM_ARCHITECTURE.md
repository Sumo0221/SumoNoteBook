## 1. 系統概述

Memory Dream 是蘇茉家族的長期記憶管理系統，負責儲存、管理和檢索所有蘇茉的對話記憶。本優化方案聚焦於三個核心目標：

| 目標 | 說明 |
|------|------|
| 衰減機制 | 記憶隨時間自動衰減，釋放儲存空間 |
| 量化評分 | 根據重要性、類型賦予記憶分數，優先保留高價值記憶 |
| 多 Agent 共享 | 讓所有蘇茉共享重要記憶，提升整體服務品質 |

## 2. 核心設計原則

### 2.1 記憶生命週期

```
┌─────────────────────────────────────────────────────────────┐
│                    記憶生命週期                               │
├─────────────────────────────────────────────────────────────┤
│  建立 → 活躍期 → 衰減期 → 歸檔/刪除                         │
│   ↑                                              ↓          │
│   └─────────────── 分數回升 ────────────────┘              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 評分公式

```
Score = (Importance × 0.50) + (Recency × 0.35) + Kind_Bonus + Merge_Bonus
```

| 參數 | 範圍 | 說明 |
|------|------|------|
| Importance | 0.0 - 1.0 | 重要性權重 50% |
| Recency | 0.0 - 1.0 | 新近度權重 35% |
| Kind_Bonus | 0 - 0.15 | 類型加成 |
| Merge_Bonus | 0 - 0.10 | 合併加成 |

## 3. 量化評分標準

### 3.1 記憶類型與初始分數

| 類型 | 初始分數 | 說明 |
|------|----------|------|
| 明確指令「記住這個」| 0.90 | 使用者明確要求記住的資訊 |
| 偏好類 | 0.85 | 老爺的個人偏好、習慣 |
| 高頻重複模式 | 0.80 | 出現 ≥3 次的重複模式 |
| 技術決策 | 0.75 | 系統架構、API 設計等技術决策 |
| 一般對話 | 0.50 | 日常對話內容 |
| 系統快照 | 0.40 | 系統狀態、環境資訊 |

### 3.2 Importance 計算

```python
importance = initial_kind_score × context_multiplier

# context_multiplier 上下文倍增器
# - 包含明確時間點（今天、明天、下週）：×1.2
# - 包含明確人物（夫人、Sunny、Ray）：×1.1
# - 包含明確地點（客廳、書房、竹北）：×1.1
# - 包含明確數字（日期、價格、數量）：×1.05
```

### 3.3 Recency 計算

```python
# 指數衰減：半衰期 30 天
recency = 0.5 ^ (days_since_creation / 30)

# 示例：
# 0 天 → 1.0
# 30 天 → 0.5
# 60 天 → 0.25
# 90 天 → 0.125
```

### 3.4 Kind_Bonus 計算

```python
kind_bonus = 0

# 根據上下文給予加成
if "絕對不能忘" in content or "一定要記住" in content:
    kind_bonus += 0.15
elif "重要的" in content or "關鍵的" in content:
    kind_bonus += 0.10
elif "提醒我" in content or "記得提醒" in content:
    kind_bonus += 0.08
```

### 3.5 Merge_Bonus 計算

```python
merge_bonus = 0

# 如果有相似記憶被合併
if merge_count >= 3:
    merge_bonus = 0.10
elif merge_count >= 2:
    merge_bonus = 0.07
elif merge_count >= 1:
    merge_bonus = 0.05
```

## 4. 衰減機制

### 4.1 衰減規則

| 維度 | 衰減方式 | 週期 | 下限 |
|------|----------|------|------|
| Recency | 指數衰減 | 即時計算 | 0.05 |
| Importance | 線性衰減 | 每 30 天 -0.02 | 0.10 |
| Kind_Bonus | 線性衰減 | 每 60 天 -0.01 | 0 |

### 4.2 衰減觸發條件

```python
def should_decay(memory):
    days = (now() - memory.created_at).days
    
    # Recency 每次存取時重新計算
    memory.recency = 0.5 ** (days / 30)
    
    # Importance 每 30 天線性衰減
    if days % 30 == 0:
        memory.importance = max(0.10, memory.importance - 0.02)
    
    return memory
```

### 4.3 記憶處理策略

| 分數區間 | 處理策略 |
|----------|----------|
| ≥ 0.70 | 活躍記憶 - 優先保留，快速檢索 |
| 0.40 - 0.69 | 冷卻記憶 - 標準儲存，定期檢視 |
| 0.20 - 0.39 | 歸檔記憶 - 壓縮儲存，延遲檢索 |
| < 0.20 | 待刪除 - 軟刪除，90 天後完全移除 |

SHARE_RULES = {

## 6. 數據結構設計

### 6.1 記憶實體 (Memory Entity)

```yaml
memory:
  id: "mem_xxxxx"              # 唯一識別碼
  agent_id: "senior_engineer"  # 所屬蘇茉
  content:                     # 記憶內容
    text: "..."                # 文字內容
    metadata: {}               # 相關metadata
  
  # 評分相關
  scoring:
    initial_kind: 0.50         # 初始類型分數
    importance: 0.50           # 重要性（會衰減）
    recency: 1.0               # 新近度（即時計算）
    kind_bonus: 0.0            # 類型加成
    merge_bonus: 0.0           # 合併加成
    total_score: 0.65          # 總分 = importance×0.50 + recency×0.35 + ...
    
  # 共享相關
  sharing:
    scope: "global"           # global / role / private
    synced: true              # 是否已同步
    synced_at: "2026-04-08T10:00:00Z"
    
  # 時間戳記
  timestamps:
    created: "2026-04-08T09:00:00Z"
    updated: "2026-04-08T10:00:00Z"
    last_accessed: "2026-04-08T12:00:00Z"
    
  # 生命週期
  lifecycle:
    status: "active"          # active / cooling / archived / deleting
    decay_count: 0            # 衰減次數
    version: 1                # 版本號
```

## 8. 實作建議

### 8.1 技術堆疊

| 層面 | 技術選擇 |
|------|----------|
| 儲存 | SQLite (本地) + Redis (緩存) |
| 索引 | PostgreSQL (全文檢索) |
| 同步 | WebSocket + Redis Pub/Sub |
| 排程 | Cron (衰減計算) |

### 8.2 實作順序

1. **Phase 1** - 評分機制
   - 實現評分公式
   - 建立分數索引
   
2. **Phase 2** - 衰減機制
   - 實現 Recency 計算
   - 實現 Importance 衰減
   - 建立 Cron 排程

3. **Phase 3** - 共享機制
   - 建立 Wing 服務
   - 實現同步邏輯
   - 實現衝突處理

### 8.3 監控指標

| 指標 | 目標 |
|------|------|
| 記憶總數 | ≤ 1000 (活躍) |
| 平均分數 | ≥ 0.50 |
| 共享記憶比例 | ≥ 10% |
| 衰減處理時間 | < 5 秒 |
