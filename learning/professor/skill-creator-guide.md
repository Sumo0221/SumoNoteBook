# 技能創作指南 - 蘇茉版

> 基於 Anthropic Skill Creator 改編，專為張家蘇茉系統設計

---

## 什麼是技能（Skill）？

技能是一組讓蘇茉能夠執行特定任務的指令集合。每個技能包含：
- **觸發條件**：什麼情況下使用這個技能
- **執行方式**：具體要做什麼
- **輸出格式**：結果怎麼呈現

---

## 技能檔案結構

```
技能名稱/
├── SKILL.md        # 必要：技能說明文件
├── scripts/        # 可選：自動化腳本
├── references/     # 可選：參考文檔
└── assets/         # 可選：素材資源
```

### SKILL.md 格式

```yaml
---
name: 技能名稱
description: 觸發條件和功能說明（最重要的部分！）
---

# 技能說明
這裡寫詳細的執行指引...
```

## 如何創建新技能

### Step 1：確定需求

回答以下問題：
1. 這個技能要做什麼？
2. 什麼情況下要觸發？（用戶說什麼話）
3. 輸入和輸出是什麼？
4. 需要什麼工具或資源？

### Step 2：撰寫 SKILL.md

**必備欄位：**
- `name`：技能識別名稱（英文，建議用 `-` 連接）
- `description`：觸發說明，要包含「什麼情況下使用」

**描述寫作技巧：**
- 具體一點，寫出典型用法
- 包含關鍵字（用戶可能說的詞彙）
- 適度「pushy」，讓蘇茉更容易觸發

**範例：**
```yaml
name: stock-price
description: 查詢股票價格。當用戶問「股票多少錢」、「股價查詢」、「某股票今天漲跌」等相關問題時使用。可以查詢台灣和美國股票。
```

### Step 3：測試與優化

1. **手動測試**：實際使用技能執行任務
2. **檢視結果**：輸出是否符合預期
3. **迭代改進**：根據結果調整 SKILL.md

### Step 4：安裝使用

將技能資料夾放到：`~/.openclaw/skills/`

## 設計技巧

### 好的描述應該：

✅ 具體的觸發情境  
✅ 包含關鍵字和同義詞  
✅ 說明輸入輸出格式  
✅ 適度詳細但不冗長

### 避免：

❌ 太抽象（「幫我處理數據」）  
❌ 太窄（只能處理一種情況）  
❌ 缺少關鍵字  
❌ 太長（超過完整說明）

## 測試案例設計

如果技能需要精確輸出，可以設計測試案例：

```json
{
  "skill_name": "example-skill",
  "evals": [
    {
      "id": 1,
      "prompt": "用戶會說的話",
      "expected_output": "預期結果"
    }
  ]
}
```

```yaml
# 股票查詢技能

## 功能
- 查詢台灣股票價格（大盤、個股）
- 查詢美國股票價格
- 顯示漲跌資訊

## 使用方式
1. 識別用戶想要的股票代碼
2. 判斷是台灣還是美國股票
3. 使用適當的資料源查詢
4. 回覆格式：股票名稱、代碼、即時價格、漲跌金額、漲跌%

## 輸出格式
📈 [股票名稱] ([代碼])
- 價格：$XXX
- 漲跌：+X.XX (+X.XX%)
- 資料來源：XXX

## 注意事項
- 如果用戶只說股票名稱，嘗試轉換為代碼
- 美國股票代碼注意大小寫
- 無法查詢時誠實告知用戶
```
