# 🛡️ 蘇茉家族反詐騙守則

> 基於「Contagious Interview（傳染性面試）」手法分析而建立
> 建立日期：2026-04-09
> 依據：Facebook 貼文分析 + 老爺架構建議

---

## 一、風險等級分類

| 等級 | 標誌 | 說明 |
|------|------|------|
| 🟢 低風險 | 正常 | 正常招募流程，無可疑跡象 |
| 🟡 中風險 | 注意 | 符合 1-2 個特徵，需要進一步調查 |
| 🔴 高風險 | 警示 | 符合 3+ 個特徵，蘇茉進入警示模式 |
| 🚨 極高風險 | 阻斷 | 確認為攻擊，直接封鎖並通知老爺 |

---

## 二、已知詐騙特徵庫（Contagious Interview 模式）

### 🔴 高風險關鍵字

| 類別 | 特徵 | 風險 |
|------|------|------|
| **薪酬異常** | 時薪 $90-110 USD、月薪 $15-20K USD | 高 |
| **快速承諾** | 面試後立刻要 clone repo 或打開附件 | 高 |
| **公司空殼化** | 網站精美但內容空洞、無實際產品 | 高 |
| **Profile 異常** | LinkedIn 新帳號、connection 少、照片模糊 | 高 |
| **技術異常** | 要求安裝特定工具、收到 .env/.pdf 附件 | 高 |
| **面試流程怪異** | AI 面試、限時、面試官聲音奇怪 | 高 |

### 🚨 已確認的攻擊手法

| 攻擊者 | 手法 | 惡意行為 |
|--------|------|----------|
| **Jayde Johnston** | Tirios.xyz 面試 | 藏有 C2 beacon，竊取 .env、API keys、加密貨幣錢包 |
| **Kateryna Horshenina** | 烏克蘭 recruiter | 推薦 AI 餐廳點餐系統，傳送惡意 PDF 附件 |

## 三、蘇茉作為第一道防線（First Line of Defense）

### 📋 處理流程

```
外部訊息（LinkedIn/Email/DM）
    ↓
蘇茉第一線處理（隔離層）
    ↓
背景檢查（自動搜尋寄件人/公司/專案）
    ↓
風險評估（對照特徵庫）
    ↓
決定行動：
  🟢 低風險 → 協助擬稿回覆
  🟡 中風險 → 提醒老爺注意
  🔴 高風險 → 通知老爺不要點擊
  🚨 極高風險 → 直接封鎖並通知
```

## 四、自動化背景審計機制

### 🔍 蘇茉自動檢查項目

| 檢查項目 | 執行方式 |
|----------|----------|
| **寄件人背景** | 搜尋該人物名稱 + "scam" / "fraud" |
| **公司資料** | 檢查公司網站、員工人數、評論 |
| **專案網址** | 使用 VirusTotal 檢測 URL |
| **附件分析** | 在沙盒環境中分析 PDF/程式碼 |
| **黑名單比對** | 對照已知詐騙特徵庫 |

### ⚠️ 觸發警示的條件

- 薪水異常高（超過市場行情 50%）
- 要求打開 PDF 附件
- 要求安裝特定工具（npm install、pip install）
- 新創公司網頁空洞
- 很快要求 Clone Repo

## 五、沙盒環境嚴格執行 (Sandbox Rule)

### 🚨 零信任原則

| 情況 | 規定 |
|------|------|
| **陌生 Repo** | 禁止在本機執行，先傳給蘇茉 |
| **PDF 附件** | 蘇茉在隔離環境中分析 |
| **安裝指令** | 由蘇茉在沙盒 Subagent 中執行 |
| **陌生連結** | 先經蘇茉用 VirusTotal 檢測 |

### 📋 執行流程

```
收到陌生 Repo/檔案
    ↓
蘇茉 Subagent 沙盒環境接手
    ↓
分析檔案結構（不執行）
    ↓
檢查可疑模式（後門、竊取資料）
    ↓
回報老爺是否有異常
```

## 六、陌生面試檢查清單

### 📝 收到新邀約時，蘇茉會對照檢查：

| # | 檢查項目 | 正常 | 可疑 |
|---|----------|------|------|
| 1 | LinkedIn Profile 新舊？ | 1年以上 | 新帳號 |
| 2 | Connection 數量？ | 500+ | 少於100 |
| 3 | 薪水是否符合市場？ | 合理範圍 | 異常高 |
| 4 | 是否很快要求下載附件？ | 否 | 是 |
| 5 | 是否很快要求 Clone Repo？ | 否 | 是 |
| 6 | 公司網站是否完整？ | 有產品/團隊 | 空殼 |
| 7 | 面試流程是否正常？ | 多輪/視訊 | 奇怪流程 |
| 8 | 是否有國外口音或 AI 面試？ | 無 | 有 |

**滿足 3+ 項 → 🔴 高風險警示模式**

## 七、自動化通知機制

### 📤 蘇茉會主動通知老爺的情況

| 情況 | 通知方式 | 緊急程度 |
|------|----------|----------|
| 🚨 發現極高風險攻擊 | 立刻 Telegram 私聊 | 最高 |
| 🔴 發現高風險特徵 | 告知並請老爺確認 | 高 |
| 🟡 發現中風險跡象 | 在對話中提醒 | 中 |

### 🔧 VirusTotal 整合

- 蘇茉可將可疑 URL 傳給 VirusTotal 檢測
- 檢測結果会顯示風險等級
